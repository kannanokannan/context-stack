# Authority Over Action

In AI-assisted delivery, agent capability is not the scarce resource. Authority over agent action is.

As agents perform delivery work — resolving tickets, drafting changes, triaging alerts — the unit of value moves from labour supplied to action governed.

The defensible position is not the largest fleet of agents. It is a stated, enforceable answer to three questions:

- Which agent may take which action?
- Who is accountable for that permission?
- How is each action audited?

Capability commoditizes. Authority does not.

## Consequence for delivery

- Pricing moves from headcount to governed outcome.
- Accountability for agent action becomes an explicit deliverable, not an assumption.
- Every agent action in a production estate is an auditable crossing.

## Governed by

- ContextOps agent-authority — definition: accountable owner, autonomy tier, least agency.
- ContextBoundary agent-authority-enforcement — runtime: identity at invocation, tier gating, audit.

Capability is bought. Authority is designed.
