# Context Stack Control Plane

`context-stack` is the coordination layer for the stack.

It owns:

- canonical terminology
- locked cross-project decisions
- repo ownership boundaries
- release checks
- agent briefing for cross-project work

It does not own:

- project-specific implementation detail
- roadmap or backlog
- temporary research notes
- active planning state

Before changing a pillar repo, check `GLOSSARY.md`, `DECISIONS.md`, `REPO_MAP.md`, and `RELEASE_CHECKLIST.md`.
